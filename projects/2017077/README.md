## Σύνοψη
Στην παρούσα αναφορά, αρχικά δίνεται μία σύντομη εισαγωγή για την θεματική του μαθήματος "Τεχνολογία Λογισμικού" του Ζ εξαμήνου. Στην συνέχεια, παραθέτονται όλες οι ζητούμενες ασκήσεις που υλοποιήσαμε και αντίστοιχα το εκπαιδευτικό υλικό που εντοπίσαμε κατά την διάρκεια διεξαγωγής του εξαμήνου. Για κάθε άσκηση που έχει υλοποιηθεί, δίνεται μία περιγραφή, link που οδηγεί στο εκάστοτε Asciinema βίντεο, links σε εργαλεία που χρησιμοποιήθηκαν και όπου είναι απαραίτητο παραθέτονται screenshots αλλά και GIFs. Παρόμοια, σε ότι έχει να κάνει με το συμμετοχικό εκπαιδευτικό υλικό, δίνεται για κάθε παραδοτέο μία περιγραφή καθώς και links στο προσωπικό αποθετήριο αλλά και την ιστοσελίδα του βιβλίου. Τέλος, υπάρχουν τα συμπεράσματα, οπού γίνεται μία ανασκόπηση των γνώσεων και ερεθισμάτων που λάβαμε μέσω της εκπαιδευτικής διαδικασίας του μαθήματος.
## Σύντομη εισαγωγή
Από τον τίτλο του μαθήματος, γίνεται εύκολα αντιληπτό ότι η θεματική έχει ως επίκεντρο το Software. Ο όρος Software όμως, απαρτίζεται απο πολλές πτυχές, αποτελώντας έτσι ένα από τα μεγαλύτερα πεδία σε ότι έχει να κάνει με την επιστήμη της πληροφορικής. Στο συγκεκριμένο μάθημα, προσεγγίσαμε το λογισμικό μέσω του command line. Στην σημερινή εποχή, η χρήση του τερματικού αποτελεί δεύτερη θέση σε ότι έχει να κάνει με αλληλεπίδραση λογισμικού, αφού το γραφικό περιβάλλον προσφέρει μία ευκολότερη διαχείριση. Θέτοντας όμως ως μέτρο σύγκρισης την ταχύτητα και όχι την ευκολία, το γραφικό περιβάλλον έρχεται σε δεύτερη μοίρα. Με την αξιοποίηση του τερματικού, δίνεται η δυνατότητα γρηγορότερης πλοήγησης καθώς και ευκολότερης διαχείρισης σε ότι έχει να κάνει με την χρήση προγραμμάτων, αρχείων κλπ. Με βάση τα παραπάνω, στα πλαίσια του μαθήματος μας ανατέθηκε η υλοποίηση αριθμού ασκήσεων με βασικό εργαλείο το terminal, φέρνοντας μας έτσι σε επαφή με αυτό τον κρυμμένο, αλλά πολύ μεγάλο κόσμο. Εκτός από την υλοποίηση των ασκήσεων, μας ανατέθηκε και η εύρεση συμμετοχικού εκπαιδευτικού υλικού με θεματικές που αφορούν γενικότερα την τεχνολογία λογισμικού. Να σημειωθεί, ότι η υλοποίηση των ασκήσεων έγινε με την χρήση του VirtualBox, δημιουργώντας ένα virtual machine με λειτουργικό σύστημα Ubuntu. Παρακάτω, παραθέτονται όλα τα ζητούμενα του μαθήματος (ασκήσεις και συμμετοχικό εκπαιδευτικό υλικό) συνοδευόμενα από την αντίστοιχη τεκμηρίωση με την χρήση καταγραφών στο Asciinema, λεκτική περιγραφή, links σε πηγές και εργαλεία που χρησιμοποιήθηκαν, screenshots και Gifs όπου κρίθηκε απαραίτητο κλπ.

#### Description:

  * [virtualenv](https://docs.python-guide.org/dev/virtualenvs/)(A tool to create isolated Python environments)

Αφού εγκαταστάθηκε το psutil, έκανα χρήση της εντολής ```pipenv shell``` έτσι ώστε να ενεργοποιηθεί το virtualenv του συγκεκριμένου project. Η πρώτη, αφορά την κατηγορία **MEMORY FUNCTIONS** και επιστρέφει στατιστικά που σχετίζονται με την χρησιμότητα της μνήμης. Η δεύτερη, αφορά την κατηγορία **SYSTEM INFORMATION FUNCTIONS** και επιστρέφει τους χρήστες που είναι συνδεδεμένοι εκείνη την στιγμή στο μηχάνημα:
 
  1. ```psutil.virtual_memory()```
  2. ```psutil.users()```
  
Αφού ολοκληρώθηκε η υλοποίηση του προγράμματος με όνομα pyscript.py, δοκίμασα να το εκτελέσω με χρήση της εντολής  ```python3 pyscript.py``` **εντός του αρχικού project**. Όπως φαίνεται και στο [Asciinema](https://asciinema.org/a/324875), το πρόγραμμα εκτελέστηκε κανονικά τυπώνοντας τα επιθυμητά αποτελέσματα στο termilal. Στην συνέχεια, με την εντολή ```exit``` αποχώρησα απο το περιβάλλον του project και κάνοντας χρήση του ```cd ..``` πήγα στο directory που βρίσκεται τοποθετημένος ο **firstfolder**. Εκεί, δημιούργησα ένα νέο φάκελο με όνομα **secondfolder** μέσα στον οποίο κάνοντας χρήση της εντολής ```cp ../firstfolder/*.py ./``` αντιγράφηκε το pyscript.py του **firstfolder**. Τέλος, δοκίμασα να τρέξω εκεί το pyscript.py αλλά όπως φαίνεται και στο βίντεο έβγαλε error σύμφωνα με το οποίο δεν αναγνωρίζεται το [psutil](https://pypi.org/project/psutil/) καθώς αυτό έχει εγκατασταθεί μόνο στο πρώτο μας project εντός του φακέλου **firstfolder**. Με αυτό τον τρόπο πετύχαμε τον επιθυμητό μας στόχο ο οποίος ήταν το isolation.
 
 
### Sixth assignment
#### Assignment: set-up cloud services
#### Deliverables: ssh to a remote machine and demonstrate your remote cli user land (e.g., email, editor, cv, code, etc)
#### [Link to Asciinema](https://asciinema.org/a/327721)
#### Description:
Για την έκτη εργασία, αποφάσισα να ασχοληθώ με το ssh πρωτόκολλο το οποίο δίνει την δυνατότητα ασφαλής σύνδεσης σε κάποιο απομακρυσμένο υπολογιστή-server μέσω ενός text-based interface. Η δυνατότητα που προσφέρει το ssh είναι πολύ χρήσιμη καθώς μπορεί να χρησιμοποιηθεί σε ποικίλες καταστάσεις όπως για παράδειγμα την σύνδεση στον υπολογιστή του σπιτιού μας ενώ βρισκόμαστε στο γραφείο. Για την δημιουργία μιας σύνδεσης χρειαζόμαστε δύο συστατικά. Το πρώτο είναι ο client και το δεύτερο ο αντίστοιχος server. Το εργαλείο που χρησιμοποίησα για την αξιοποίηση του ssh protocol είναι το [OpenSSH](https://www.openssh.com/). Συγκεκριμένα, τα ubuntu έχουν απο default εγκατεστημένο το SSH client οπότε χρειάστηκε να εγκαταστήσω μόνο το SSH server χρησιμοποιόντας την εντολή  ```sudo apt-get install openssh-server```. Αφού γίνει το installation, κάνοντας χρήση της εντολής ```sudo service ssh status``` σου δίνεται η δυνατότητα να δεις την τρέχουσα κατάσταση του server. Απαραίτητη προυπόθεση για την πραγματοποίηση μιας σύνδεσης είναι ο υπολογιστής-server που επιθυμούμε να συνδεθούμε να έχει εγκατεστημένο το εργαλείο openssh-server. Με την εντολή  ```sudo nano /etc/ssh/sshd_config ``` σου δίνεται η δυνατότητα να κάνεις configuration τον server σου όπως για παράδειγμα να επιλέξεις το port μέσω του οποίο θα πραγματοποιείται η σύνδεση, την μέθοδο που θα γίνεται για το authentication του χρήστη που επιθυμεί να συνδεθεί κλπ. Για την σύνδεση μου, επέλεξα να χρησιμοποιήσω το default port το οποίο είναι το 22. Όπως φαίνεται και στο [Asciinema](https://asciinema.org/a/327721), συνδέθηκα στον server τοπικά με χρήση της εντολής ssh localhost. Εαν θέλουμε να συνδεθούμε σε ενα μηχάνημα εκτός του τοπικού μας δικτύου τότε η σύνταξη της εντολής είναι ```ssh your_username@host_ip_address```.  ### Seventh assignment
#### Assignment: configure a custom window manager
#### Deliverables: try different wm and configure one to fit your needs
#### [Link to Asciinema](https://asciinema.org/a/333760)
#### Description:
Λόγω της αλλαγής στον τρόπο εξέτασης του μαθήματος (επιπλέον άσκηση και συμμετοχικό υλικό), σε ότι αφορά την επιπλέον άσκηση, αποφάσησα να ασχοληθώ με το [i3 window manager](https://i3wm.org/docs/userguide.html). Με την χρήση ενός **wm** μπορείς να δημιουργήσεις ένα περιβάλλον βασισμένο περισότερο στην χρήση του πληκτρολογίου, αυξάνοντας κατά αυτό τον τρόπο την ταχύτητα εργασίας,πλοήγησης κλπ. Οι δυνατότητες είναι απεριόριστες λόγω του μεγάλου πλήθους επιλογών τις οποίες μπορεί να χρησιμοποιήσει κάποιος φέρνοντας το **wm** στα μέτρα του. Το [i3](https://i3wm.org/docs/userguide.html), αποτελεί έναν από τους πιο διαδεδομένους window managers, διαθέτει ένα πολύ επεξηγηματικό [documentation](https://i3wm.org/docs/userguide.html), με την χρήση του οποίου μπορείς να κάνεις "μαγικά" τροποποιόντας το configuration file. Η εγκατάσταση του [i3](https://i3wm.org/docs/userguide.html) είναι απλή και γίνεται με την χρήση της εντολής ```sudo apt-get install i3```. Πριν τις προσθήκες/αλλαγές στο configuration file, το i3 είναι αρκετά απλό και προσφέρει μόνο βασικές λειτουργίες. Αυτό γίνεται επίτηδες, αφού είναι ευθύνη του εκάστοτε χρήστη να το τροποποιήσει έτσι ώστε να καλύπτει τις ανάγκες του. Παρακάτω, περιγράφω τις προσθήκες/αλλαγές που έκανα στο configuration file του [i3](https://i3wm.org/docs/userguide.html) όπως φαίνονται και στο [asciinema](https://asciinema.org/a/333760), δημιουργόντας έτσι ένα περιβάλλον βασισμένο στις προτιμήσεις μου, αποφεύγοντας την εκτεταμένη χρήση του touchpad.

1. Με την εντολή ```bindsym Print exec gnome-screenshot -i``` δίνεται η δυνατότητα λήψης και αποθήκευσης στιγμιοτύπου της οθόνης(screenshot) με χρήση του κουμπιού **PrtScn**.
2. Με την εντολή ```bindsym $mod+shift+x exec i3lock``` δίνεται η δυνατότητα κλειδώματος της οθόνης με χρήση του συνδιασμού των κουμπιών **$mod+shift+x**. Να σημειωθεί ότι με το install του [i3](https://i3wm.org/docs/userguide.html) επιλέγεις την πρώτη φορά ποιο θέλεις να είναι το **$mod key**. Στην περιπτωσή μας, ως **$mod key** επέλεξα το **Alt**.
3. Εγκατέστησα το [feh](https://feh.finalrewind.org/) (δεν φαίνεται στο βίντεο) και με την χρήση της εντολής ```exec_always feh --bg-scale /home/chris/Desktop/wallpaper.jpeg``` όρισα ένα wallpaper στον περιβάλλον αυξάνοντας έτσι την αισθητική του.
4. Οι εντολές ```exec gnome-terminal```, ```exec firefox```, ```exec rhythmbox``` έχουν ως αποτέλεσμα κάθε φορά που κάνω login να ανοίγει αυτόματα ένα τερματικό, ο firefox και το πρόγραμμα αναπαραγωγής μουσικής rhythmbox.
5. Προσθέτω στην εντολή που ορίζει τον συνδιασμό των κουμπιών για άνοιγα του **dmenu** τις παραμέτρους ```-nb red -sb green``` τροποποιώντας έτσι χρωματικά το **dmenu**.
6. Στο [i3](https://i3wm.org/docs/userguide.html), έχεις διαθέσιμα **workspaces** αριθμημένα απο το ένα μέχρι το δέκα. Εντός του κάθε **workspace**, μπορείς να ανοίξεις όσα παράθυρα θέλεις ( π.χ terminal,firefox κλπ.). Τα **workspaces** με τα οποία δουλεύεις φαίνονται κάτω αριστερά στην οθόνη. Όπως φαίνεται στο [asciinema](https://asciinema.org/a/333760), γίνεται τροποποίηση ονομάτων των **workspaces** **1**, **2**, **3** σε **Terminals**, **Music**, **Browsing**. Με αυτό τον τρόπο, γίνεται μία κατηγοριοποίηση σύμφωνα με την οποία στο **workspace** με όνομα **Terminals** μπορούμε να ανοίγουμε μόνο τερματικά, στο **workspace** **Music** ότι αφορά αναπαραγωγή μουσικής και στο **Browsing** ότι έχει να κάνει με το διαδίκτυο, χωρίς να χρειάζεται να θυμόμαστε σε ποιό **workspace** είναι τί, κάτι το οποίο θα συνέβαινε αν τα **workspaces** είχαν αριθμούς για ονόματα.
7. Με τις εντολές ```assign [class = "Rhythmbox"] Music```, ```assign [class = "Gnome-terminal"] Terminals```, ```assign [class = "Firefox"] Browsing``` ορίζουμε σε ποιό **workspace** θέλουμε να ανοίγει κάθε φορά το αντίστοιχο πρόγραμμα που καλούμε. Στο παράδειγμα μας, ορίζουμε κάθε φορά που ανοίγουμε ένα τερματικό αυτό να ανοίγει στο **workspace** **Terminals**, κάθε φορά που ανοίγουμε το πρόγραμμα rhythmbox αυτό να ανοίγει στο **workspace** **Music** και τέλος κάθε φορά που ανοίγουμε το firefox αυτό να ανοίγει στο **workspace** **Browsing**.

Παρακάτω, φαίνονται δύο gif και μία εικόνα. Στο πρώτο gif, κάνω login και πλοηγούμαι στο i3 πρίν τις προσθήκες/αλλαγές στο configuration file. Στο δεύτερο gif, κάνω login και πλοηγούμαι στο i3 μετά τις προσθήκες/αλλαγές στο configuration file. Παρατηρούμε, πόσο διαφορετικό έγινε το i3 μετά τις προσθήκες/αλλαγές που αναφέραμε παραπάνω. Στο δεύτερο gif, βλέπουμε ότι κατευθείαν μετά το login ανοίγουν αυτόματα ένα τερματικό, το rhythmbox και ο firefox, το καθένα στο workspace που έχουμε ορίσει μέσα στο configuration file. Στην συνέχεια, ανοίγουμε το **workspace** **4** και μέσω αυτού ανοίγουμε το firefox και ένα terminal παρατηρώντας ότι θα ανοίξει το καθένα στο κατάλληλο **workspace** (το τερματικό στο **Terminals** **workspace** και το firefox στο **Browsing**. Επίσης, παρατηρούμε το wallpaper και τα διαφορετικά χρώματα που έχει το **dmenu**, καθώς επίσης και το κλείδωμα της οθόνης με χρήση του συνδιασμού των κουμπιών **$mod+shift+x**. Τέλος, στην εικόνα βλέπουμε την επιλογή που δίνεται για screenshot, κάτι το οποίο δεν ήταν δυνατό πρίν τις αλλαγές/προσθήκες στο configuration file.
